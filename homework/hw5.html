Question # Answer
1 In order to include Bootstrap in our pages/sites/apps, we will use Bootstrap's CDN using the following code.

<link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/4.0.0/css/bootstrap.min.css">
2 CDN stands for content delivery networks. It is a system of distributed servers that deliver pages and web content (like boostrap's css and js files) to users based on geographic locations in order to reduce web page loading time.
3 The most beneficial reason for using CDN is for speed. Big sites that provide these CDN usually have better servers than we do so it'll be faster for users to get the file via CDN rather than our server. Browsers also have limits on number of concurrent assets per website, which means using CDN would not be included in the limit as it is provided by another website. Last but not least, there is a good chance that ISP has already cached files because of popularity of the CDN.
4 Breakpoints in RWD are points at which website's content will respond to provide the user with the best possible layout to consume the information.
5 Bootstrap primarily uses the following breakpoint values.

// Extra small devices (portrait phones, less than 576px)
// No media query since this is the default in Bootstrap

// Small devices (landscape phones, 576px and up)
@media (min-width: 576px) { ... }

// Medium devices (tablets, 768px and up)
@media (min-width: 768px) { ... }

// Large devices (desktops, 992px and up)
@media (min-width: 992px) { ... }

// Extra large devices (large desktops, 1200px and up)
@media (min-width: 1200px) { ... }
6 In Bootstrap, fixed width container have a specific fixed width based on the different breakpoint values. Full width container, on the other hand, will be take up the entire width without relying on breakpoint values.
7 Bootstrap (and many RWD implementations) break the page up into 12 columns.
8 In Bootstrap, row provide the columns a place to live and should always be in a container.
